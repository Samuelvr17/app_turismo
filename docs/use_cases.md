# Casos de uso

La siguiente tabla detalla los casos de uso derivados de los requerimientos funcionales RF1–RF8.

| ID | Nombre | Actores | Propósito | Disparador | Precondiciones | Flujo principal | Flujo alterno / Excepciones | Postcondiciones |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| CU1 | Registro de usuarios | Turista | Registrar un nuevo usuario en la aplicación. | El turista solicita crear una cuenta. | El usuario no tiene una cuenta activa y dispone de conexión a internet. | 1. El turista ingresa al módulo de registro.<br>2. Proporciona email, nombre completo y contraseña.<br>3. El sistema valida formato y unicidad del email.<br>4. El sistema crea el registro en la base de datos y guarda la sesión. | Si los datos no son válidos o el email ya existe, el sistema muestra el error y solicita corrección. | La cuenta queda creada y la sesión iniciada. |
| CU2 | Inicio y cierre de sesión | Turista | Permitir acceder y salir de la aplicación con credenciales válidas. | El turista ingresa sus credenciales en la pantalla de acceso. | El usuario está registrado y conoce sus credenciales. | 1. El turista ingresa email y contraseña.<br>2. El sistema verifica las credenciales.<br>3. El sistema carga la información personalizada y habilita la navegación.<br>4. El turista puede cerrar sesión desde el menú, liberando la sesión actual. | Si la autenticación falla, el sistema informa el error e invita a reintentar o recuperar contraseña. | Sesión autenticada activa o sesión cerrada según la acción realizada. |
| CU3 | Visualizar mapa de zonas de peligro | Turista | Consultar zonas de riesgo sobre un mapa interactivo. | El turista abre el mapa de seguridad. | El dispositivo tiene permisos de localización activos. | 1. La aplicación solicita la ubicación actual.<br>2. Obtiene datos de zonas de peligro desde el servicio correspondiente.<br>3. Muestra el mapa con las zonas resaltadas. | Si la ubicación no está disponible, se muestra mensaje y se propone habilitar permisos o introducir ubicación manual. | Zonas de peligro visibles en el mapa según la ubicación disponible. |
| CU4 | Generar rutas turísticas seguras | Turista | Presentar rutas sugeridas y sus actividades asociadas. | El turista solicita rutas seguras para una zona. | El mapa está cargado y la app puede acceder al catálogo de rutas. | 1. El turista selecciona la ubicación o vereda de interés.<br>2. El sistema consulta rutas y actividades disponibles.<br>3. Muestra rutas recomendadas con detalles de actividades, tiempos y ubicación. | Si no existen rutas para la zona, el sistema informa y sugiere filtros alternativos. | Rutas seguras visibles, disponibles para navegación o guardado. |
| CU5 | Registrar reportes geolocalizados | Turista | Informar incidentes o riesgos en una ubicación determinada. | El turista elige crear un nuevo reporte. | El usuario está autenticado y el dispositivo tiene permisos de localización. | 1. El turista ingresa descripción y tipo de incidente.<br>2. El sistema captura coordenadas actuales o seleccionadas.<br>3. El sistema valida la información y almacena el reporte. | Si falla la geolocalización, el sistema permite ingresar la ubicación manual o guardar el borrador para sincronizar luego. | Reporte almacenado y disponible para otros usuarios tras sincronización. |
| CU6 | Recomendar actividades con IA | Turista, Servicio de IA | Ofrecer recomendaciones personalizadas según preferencias. | El turista completa el cuestionario de intereses. | El usuario está autenticado y el servicio de IA se encuentra disponible. | 1. El turista responde el cuestionario o actualiza sus preferencias.<br>2. La app envía los datos al servicio de IA.<br>3. El servicio procesa y devuelve actividades recomendadas.<br>4. La app muestra las sugerencias priorizadas. | Si el servicio de IA no responde, la app notifica la indisponibilidad y ofrece actividades generales. | Recomendaciones personalizadas registradas para consulta futura. |
| CU7 | Consultar clima en tiempo real | Turista, Servicio meteorológico | Proporcionar información climática actual para planificar actividades. | El turista abre la sección de clima o una ruta que requiere datos meteorológicos. | El dispositivo tiene conexión a internet y servicios de ubicación habilitados. | 1. La app obtiene la ubicación vigente.<br>2. Solicita datos al servicio meteorológico externo.<br>3. Muestra temperatura, pronóstico y alertas relevantes. | Si el servicio no responde, se muestra el último dato almacenado y un aviso de actualización pendiente. | Clima actualizado mostrado en la interfaz del usuario. |
| CU8 | Visualizar información en realidad aumentada | Turista, Servicios AR | Superponer información contextual en la vista AR. | El turista activa la vista de realidad aumentada desde el mapa. | El dispositivo es compatible con AR y tiene permisos de cámara y ubicación. | 1. La app verifica compatibilidad y permisos AR.<br>2. Carga datos de zonas de peligro e información contextual.<br>3. Superpone elementos informativos sobre la vista de cámara en tiempo real. | Si los permisos no están concedidos o el dispositivo no soporta AR, el sistema informa y ofrece la vista estándar. | Información de seguridad mostrada en AR para el entorno observado. |
